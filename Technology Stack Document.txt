Technology Stack Document
Unified Browser-Based Multiplayer Game Suite with ML-Driven IQ Monitoring
________________________________________
1. Purpose
This document defines the complete technology stack used to build the Unified Multiplayer Game Suite with Machine Learning–based IQ Monitoring. It explains what technologies are used, why they are chosen, and how they align with academic requirements, system design, and scalability goals.
This document is intended for:
•	Academic submission and viva
•	Developer onboarding
•	Architecture and technology justification
________________________________________
2. Guiding Principles for Tech Selection
•	Browser-first accessibility
•	Deterministic and testable core logic
•	Explainable and ethical ML usage
•	Minimal but powerful toolchain
•	Clear separation of training and inference
•	Easy extensibility

7.4 ML Inference Layer
Option A: Client-Side Inference (Preferred for Academics)
•	JavaScript-based inference
•	Pre-trained model exported to JSON / ONNX
•	Offline and privacy-friendly
Option B: Server-Side Inference (Scalable)
•	Python-based REST API
•	Centralized model updates
•	Suitable for real-time multiplayer
Design Rule
ML inference influences difficulty and analytics only, never game rules.
