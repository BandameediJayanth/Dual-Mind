Product Requirements Document (PRD)
DualMind — Cognitive Gaming Platform with ML Analytics
________________________________________

1. Purpose

This PRD defines the vision, scope, features, and requirements for DualMind — a browser-based cognitive gaming platform with Machine Learning analytics. It supports:
•	Developer onboarding and collaboration
•	Academic submission and evaluation
•	Clear alignment between product goals and technical design

DualMind combines classic games with explainable ML to evaluate and visualize player cognitive performance in a non-clinical, transparent, and ethical manner.
________________________________________

2. Product Vision

A premium web gaming platform where every game you play reveals something about how you think. DualMind uses machine learning to analyze decision-making patterns — speed, accuracy, strategy, consistency — and provides players with a cognitive performance profile that improves over time.
________________________________________

3. Objectives

•	Provide a unified platform for 10 classic games with consistent UI
•	Capture real-time gameplay behavior as structured data
•	Apply ML models to predict skill tiers and performance indices
•	Display cognitive insights after every game
•	Scale to support 10,000+ concurrent users
•	Maintain ethical, transparent, non-clinical analytics
________________________________________

4. Games (Implemented)

The following 10 games are fully implemented and integrated with the ML pipeline:

  Strategy Games:
    1. Tic Tac Toe — Pattern recognition, optimal play
    2. Four in a Row — Strategic planning, column control
    3. Checkers — Multi-step thinking, capture optimization

  Territory Games:
    4. Dots & Boxes — Edge selection strategy, area control
    5. Color Wars — Territorial expansion, risk assessment
    6. Sea Wars — Grid-based combat, probabilistic reasoning

  Memory & Language Games:
    7. Memory Match — Recall accuracy, error repetition
    8. Word Chain — Vocabulary breadth, association speed

  Chance & Decision Games:
    9. Ludo — Risk handling under randomness
    10. Snake & Ladders — Adaptability to chance events

Each game emits standardized events (game:move, game:end) for ML feature extraction.

6. Key Features

6.1 Core Platform
•	Single-page React application (no page reloads)
•	Premium dark/light theme with smooth animations
•	Responsive design (mobile, tablet, desktop)
•	Page loader, cursor follower, particle effects
•	Zero-config: works in any modern browser

6.2 Game Experience
•	Consistent game-to-game UI via GameView component
•	Result modal with play again / quit options
•	ML prediction displayed in result modal
•	Back to Games navigation from any game
•	Per-game difficulty indicators

6.3 ML Analytics (Current)
•	Data Collection:
    - Decision time per move
    - Move accuracy and error rate
    - Pattern success rate
    - Strategic depth and consistency
    - 15+ total features per session

•	Models:
    - Skill Predictor: Random Forest classifier → 5 tiers
    - Performance Estimator: Random Forest regressor → 0-100 score
    - Trend Analyzer: Linear regression on session history

•	Outputs (shown in result modal):
    - Skill tier badge (Novice → Expert) with confidence %
    - Performance index bar (0-100) with animation
    - Prediction source indicator (ML Model vs Rule-based)

6.4 Backend API
•	FastAPI + Uvicorn (async, production-ready)
•	Auto-generated Swagger docs at /docs
•	TTL cache for prediction results (60s)
•	Rate limiting (100 req/min per IP)
•	Health check and monitoring endpoints
________________________________________

7. Functional Requirements

7.1 Gameplay
•	Deterministic state machines per game
•	Valid move enforcement
•	Win/draw/loss detection
•	Event-based communication (game:move, game:end)
•	Game engines independent of ML predictions

7.2 ML Pipeline
•	Client-side feature extraction (FeatureExtractor.js)
•	API communication with fallback (MLClient.js)
•	React context for ML state management (MLProvider.jsx)
•	Session history in localStorage
•	Graceful degradation when Python service is offline

7.3 Backend
•	Model loading at startup (ModelPool singleton)
•	Cached predictions (Cache-Aside pattern)
•	Rate-limited API access
•	Gzip response compression
•	CORS support for frontend communication
________________________________________

8. Non-Functional Requirements

Performance:
•	Frontend: <2s build time, <3s initial load
•	Backend: ~8,000 req/s capacity (4 Uvicorn workers)
•	ML inference: <50ms per prediction
•	Cache hit: <1ms response time

Reliability:
•	Fallback predictions when Python service is offline
•	Graceful error handling in all components
•	Zero build warnings

Scalability:
•	Horizontal scaling via additional Uvicorn workers
•	Stateless API design
•	Cache-first architecture

Ethics:
•	Non-clinical framing ("Performance Index", not "IQ")
•	No personal or biometric data
•	All analytics locally stored
•	Feature importance transparency

11. Risks & Mitigation

Risk                          Mitigation
Performance misinterpretation  Clear non-clinical labeling, explanations
Model trained on synthetic data  Planned real data collection pipeline
Backend overload              Rate limiting, caching, horizontal scaling
ML service downtime           JavaScript fallback predictions
Feature drift over time       Periodic model retraining pipeline

13. Contributor Guide

Getting Started:
  1. Clone: git clone https://github.com/BandameediJayanth/Dual-Mind.git
  2. Frontend: npm install && npm run dev
  3. Backend: cd python && pip install -r requirements.txt
  4. Start API: uvicorn server:app --port 8000 --reload
  5. Open http://localhost:5173

Key Directories:
  src/components/     — React UI components
  src/js/games/       — Vanilla JS game engines
  src/js/ml/          — ML client and feature extraction
  python/server.py    — FastAPI backend
  python/ml/          — ML models and processors
  python/models/      — Trained .pkl model files
